### Current Design Token Architecture (globals.css)

The project uses **OKLCH color model** with CSS custom properties:
- `:root` block = light mode tokens
- `.dark` block = dark mode tokens
- `@theme inline` block maps CSS vars to Tailwind utilities (e.g., `--color-primary: var(--primary)`)

**CRITICAL:** New Copa color tokens must follow the same OKLCH pattern. Convert hex colors:
- `#009B3A` (verde) ŌåÆ approximate `oklch(0.55 0.17 152)` ŌĆö verify with converter
- `#FFDF00` (amarelo) ŌåÆ approximate `oklch(0.91 0.20 100)` ŌĆö verify with converter
- `#002776` (azul) ŌåÆ approximate `oklch(0.28 0.14 265)` ŌĆö verify with converter

Use an OKLCH converter tool or test visually to get accurate values. The hex values are the source of truth for visual appearance.
